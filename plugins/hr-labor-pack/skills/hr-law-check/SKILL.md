# /hr-law-check - 法改正リサーチスキル

人事労務拡張パック: 3段構えの法改正追従システム（第3段 - 手動徹底リサーチ）

## 概要

最新の法改正・料率変更情報をWebSearchで徹底調査し、rate-master.jsonとの差分を検出。
変更があればrate-master.jsonの更新案を生成する。

## 使い方

```bash
# 全項目の法改正チェック
/hr-law-check

# 特定項目のみ
/hr-law-check 社会保険料率
/hr-law-check 所得税
/hr-law-check 雇用保険
/hr-law-check 最低賃金
```

## 実行フロー

### Phase 1: 現在の料率マスタ読み込み

1. rate-master.jsonを読み込む
2. 各項目のlastUpdatedとeffectiveDateを確認
3. チェック対象リストを作成

### Phase 2: 最新情報のWebSearchリサーチ

以下の検索を並列実行:

```
WebSearch("社会保険料率 ${currentYear}年度 健康保険 厚生年金 最新")
WebSearch("雇用保険料率 ${currentYear}年度 改定")
WebSearch("源泉徴収税額表 ${currentYear}年 変更点")
WebSearch("最低賃金 ${currentYear}年 改定")
WebSearch("年末調整 ${currentYear}年 法改正 変更点")
WebSearch("子ども子育て支援金 ${currentYear}年度 料率")
```

### Phase 3: 差分検出

rate-master.jsonの値とリサーチ結果を比較:

1. **料率の変更**: 数値の差異を検出
2. **新制度の追加**: rate-master.jsonに未記載の項目
3. **制度の廃止/変更**: 既存項目の意味変更
4. **適用日の確認**: いつから新料率が適用されるか

### Phase 4: レポート出力

差分があった場合:
```markdown
## 法改正チェック結果

### ⚠ 変更検出
| 項目 | 現在のマスタ | 最新情報 | 適用日 | 出典 |
|------|------------|---------|--------|------|
| 健康保険料率 | 9.9% | X.X% | 2026-03-01 | [URL] |

### ✅ 変更なし
- 厚生年金: 18.3%（変更なし）
- ...

### 更新案
以下のrate-master.json更新をAskUserQuestionで提案:
- 変更箇所のdiff表示
- ユーザー承認後にEditで更新
```

差分がなかった場合:
```
✅ 全項目が最新です（最終確認: YYYY-MM-DD）
→ rate-master.jsonの nextReviewDate を30日後に更新
```

### Phase 5: rate-master.json更新

ユーザー承認後:
1. 変更箇所をEditで更新
2. lastUpdatedを現在日時に更新
3. nextReviewDateを次の重要改定月に設定
4. sourcesに新しい出典URLを追加

## 自動リサーチの信頼性担保

### 3重クロスチェック
1. **公式サイト優先**: 厚労省、国税庁、協会けんぽの公式情報を最優先
2. **複数ソース照合**: 最低2つの独立したソースで確認
3. **人間承認必須**: 料率変更は必ずユーザーに確認してから反映

### 出典の信頼度ランク
| ランク | ソース | 扱い |
|--------|--------|------|
| A（確定） | 厚労省、国税庁、年金機構、協会けんぽ | そのまま採用 |
| B（高信頼） | 社労士事務所、税理士法人のブログ | A情報で裏取り |
| C（参考） | 一般メディア、比較サイト | 単独では採用しない |

## 重要改定スケジュール

| 月 | 確認項目 |
|----|---------|
| 1月 | 源泉徴収税額表の改定適用 |
| 3月 | 健康保険・介護保険料率の改定 |
| 4月 | 雇用保険料率・子育て支援金の改定 |
| 7月 | 算定基礎届（4-6月報酬の届出） |
| 10月 | 最低賃金の改定 |
| 11月 | 年末調整の法改正確認 |

## 注意事項

- このスキルはrate-master.jsonの**読み取り+更新提案**のみ行う
- 実際のシステムコードの修正は行わない（それはhr-payroll-validatorの役割）
- 料率の変更は必ずユーザーの明示的な承認を得てから反映する
- WebSearchの結果が矛盾する場合は、公式サイトの情報を優先する
