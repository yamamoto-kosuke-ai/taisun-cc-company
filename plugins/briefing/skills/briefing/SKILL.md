# Morning Briefing Skill

## 概要
セッション開始時に秘書が自動生成するデイリーブリーフィング。
手動で `/briefing` を実行しても生成可能。

## 生成フロー

```
[秘書が起動] → [情報収集] → [ブリーフィング生成] → [ユーザーに提示]
```

## 手順

### 1. 情報収集（ローカルのみ・API不要）

以下の情報源を**読み取り専用**でチェックする:

#### a. 未完了TODO
- `.company/secretary/todos/` 内の最新ファイルを読む
- `- [ ]` の未完了タスクを抽出

#### b. 昨日の作業サマリー
- `SESSION_HANDOFF.md` があれば読む
- `.company/secretary/inbox/` の最新ファイルを読む

#### c. プロジェクト状態
- `git log --oneline -5` で直近のコミットを確認
- `git status` で未コミットの変更を確認

#### d. メモリからの知見
- `.claude/memory/` 内の最近更新されたファイルを確認
- 前セッションで記録した学びや課題を抽出

### 2. ブリーフィング生成

以下の形式で秘書の口調で出力する:

```
おはようございます！今日のブリーフィングです。

## 未完了タスク
- [ ] タスク1 | 優先度: 高
- [ ] タスク2 | 優先度: 通常

## 昨日のサマリー
- 〇〇を実装しました
- △△の調査が途中です

## プロジェクト状態
- 直近コミット: feat: xxx
- 未コミット変更: 3ファイル

## 今日の提案
- 優先度「高」のタスクから着手しませんか？
- 昨日途中だった△△の調査を続けましょうか？
```

### 3. 出力ルール

- **秘書の口調**で出力（丁寧だが親しみやすい）
- 情報がない項目はスキップ（「なし」と書かない）
- 提案は**最大3つ**に絞る
- 全体で**15行以内**に収める（長すぎると読まれない）

## 注意事項

- Web検索は**しない**（トークン節約・高速化のため）
- ファイルの書き込みは**しない**（読み取り専用）
- APIコールは**しない**（コストゼロ）
- 情報がほぼない場合は「特に共有事項はありません。何から始めましょうか？」で終了
