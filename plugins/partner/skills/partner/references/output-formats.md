# 出力フォーマット別ガイド

各ツールへの出力時に、そのツールに最適化された形式で出力するためのガイド。

---

## 1. Chatwork

### API仕様
- エンドポイント: `https://api.chatwork.com/v2`
- 認証: `X-ChatWorkToken` ヘッダー
- 認証情報: プロジェクトルートの `.chatwork-config.json` から `api_token` を読み取る

### メッセージ記法
```
[info][title]タイトル[/title]本文[/info]
[To:アカウントID]名前さん
[code]コードブロック[/code]
[hr] ← 水平線
```

### 利用可能なAPI操作
| 操作 | エンドポイント | メソッド |
|------|-------------|---------|
| ルーム一覧 | /rooms | GET |
| メッセージ取得 | /rooms/{id}/messages | GET |
| メッセージ送信 | /rooms/{id}/messages | POST |
| タスク一覧 | /rooms/{id}/tasks | GET |
| タスク作成 | /rooms/{id}/tasks | POST |
| ファイルアップロード | /rooms/{id}/files | POST (multipart) |
| 自分の情報 | /me | GET |

### セキュリティルール
- **メッセージ送信前**: 必ず送信内容とルーム名を表示し、ユーザーの承認を得る
- **ファイルアップロード前**: ファイル名とルーム名を表示し、ユーザーの承認を得る
- **一括送信禁止**: 複数ルームへの同時送信は行わない（1ルームずつ確認）

### 出力テンプレート（報告文）
```
[info][title]【報告】{{TITLE}}[/title]
{{BODY}}

対応事項:
{{ACTION_ITEMS}}
[/info]
```

### 出力テンプレート（共有）
```
[info][title]{{TITLE}}[/title]
{{SUMMARY}}

▼ 詳細
{{DETAIL_URL_OR_CONTENT}}
[/info]
```

## 2. Google スライド

### API仕様
- 認証: プロジェクトルートのサービスアカウントJSONキーを使用
- スコープ: `https://www.googleapis.com/auth/presentations`, `https://www.googleapis.com/auth/drive.file`

### 出力方針
Claude Codeからスライドを作成する際は、Google Slides APIで直接操作する。

### スライド構成テンプレート
出力時は以下の構造で整理してからAPI呼び出し:

```
スライド1: 表紙
  - タイトル
  - サブタイトル（日付、作成者等）

スライド2: 目次 / アジェンダ
  - 箇条書きで3〜5項目

スライド3〜N: 本文
  - 各スライドに見出し1つ + ポイント3〜5つ
  - 1スライド1メッセージの原則

スライドN+1: まとめ / ネクストアクション
  - 箇条書きで要点整理
```

### テキスト量の目安
- タイトル: 20文字以内
- 箇条書き1項目: 40文字以内
- 1スライドの箇条書き: 最大5項目
- 補足説明が長い場合: ノート（スピーカーノート）に記載

## 3. Google ドキュメント

### API仕様
- 認証: サービスアカウントJSONキー
- スコープ: `https://www.googleapis.com/auth/documents`, `https://www.googleapis.com/auth/drive.file`

### 出力方針
見出し構造を持った文書として作成。

### 文書構成テンプレート
```
タイトル（見出し1）
├── セクション1（見出し2）
│   ├── サブセクション（見出し3）
│   └── 本文
├── セクション2（見出し2）
│   └── 本文
└── 参考情報
```

### 書式ルール
- 見出しレベルは3段階まで
- 表はGoogle Docs APIのtable insertで作成
- 箇条書きはbullet listとして挿入
- 強調はbold、補足はitalic

## 4. Google スプレッドシート

### API仕様
- 認証: サービスアカウントJSONキー
- スコープ: `https://www.googleapis.com/auth/spreadsheets`, `https://www.googleapis.com/auth/drive.file`

### 出力方針
構造化データはスプレッドシートに出力。

### 出力ルール
- 1行目はヘッダー（太字、背景色付き）
- データは2行目以降
- 列幅は内容に応じて自動調整
- 数値データは右寄せ
- 日付は `YYYY/MM/DD` 形式

### よくある出力パターン
```
比較表:     | 項目 | A社 | B社 | C社 |
タスク一覧:  | タスク | 担当 | 期限 | 状態 |
調査結果:   | 調査項目 | 結果 | 出典 | メモ |
経費一覧:   | 日付 | 項目 | カテゴリ | 金額 |
```

## 5. Google Drive

### API仕様
- 認証: サービスアカウントJSONキー
- スコープ: `https://www.googleapis.com/auth/drive.file`

### 操作方針
- **新規作成**: 指定フォルダにファイルを作成
- **検索**: ファイル名やフォルダ名で検索
- **読み取り**: ファイルの内容を取得
- **削除は行わない**: 安全のためファイル削除機能は使用しない

### フォルダ構成の提案
サービスアカウントに共有されたフォルダ内で作業:
```
共有フォルダ/
├── partner-output/        ← AIが作成したファイルの出力先
│   ├── スライド/
│   ├── ドキュメント/
│   └── スプレッドシート/
└── （ユーザーの既存フォルダ）
```

## 6. コピペ用テキスト出力（ツール連携なし）

外部ツール連携が未設定の場合、コピペしやすい形式で出力する。

### スライド向けコピペ形式
```
━━━ スライド 1: 表紙 ━━━
タイトル: XXXXX
サブタイトル: XXXXX

━━━ スライド 2: 背景 ━━━
• ポイント1
• ポイント2
• ポイント3

━━━ スライド 3: ... ━━━
```

### スプレッドシート向けコピペ形式
タブ区切り（TSV）で出力すると、スプレッドシートにそのまま貼れる:
```
項目\tA社\tB社\tC社
価格\t1000\t1200\t800
機能\t○\t△\t○
```
（\t は実際のタブ文字に置換して出力）

### Chatwork向けコピペ形式
Chatwork記法をそのままテキストとして出力:
```
[info][title]...[/title]...[/info]
```

## セキュリティ共通ルール

1. **外部サービスへの送信・書き込み前は必ずユーザー確認**
2. **認証情報はファイル内容を表示しない**（存在確認のみ）
3. **ファイル削除は一切行わない**（Google Drive、ローカルともに）
4. **認証ファイルのパスはプロファイルに保存しない**（都度検出する）
5. **APIエラー時はエラー内容をユーザーに報告し、コピペ形式にフォールバック**
