# 人事労務ドメイン知識ルール

## 概要
人事・経理・労務システム開発時にAIが参照すべき業務ルール。
コード生成・レビュー時にこのルールに基づいて正確な実装を行う。

---

## 給与計算の基本フロー

```
勤怠データ → 総支給額計算 → 控除計算 → 差引支給額（手取り）→ 振込データ
```

### 総支給額の構成
1. **基本給** - 月額固定
2. **時間外手当（残業代）** - 法定計算式あり
3. **深夜手当** - 22時〜5時は25%以上割増
4. **休日手当** - 法定休日は35%以上割増
5. **通勤手当** - 非課税限度額あり（月15万円まで）
6. **その他手当** - 役職手当、家族手当、住宅手当等

### 残業代の法定計算式
```
残業代 = 1時間あたりの賃金 × 割増率 × 残業時間

1時間あたりの賃金 = 月給 ÷ 月平均所定労働時間
月平均所定労働時間 = 年間所定労働日数 × 1日の所定労働時間 ÷ 12

割増率:
- 法定時間外（60h以内）: 1.25
- 法定時間外（60h超）: 1.50
- 法定休日: 1.35
- 深夜: +0.25（他の割増に加算）
```

### 控除の種類と順序
1. **健康保険料** - 標準報酬月額 × 料率 ÷ 2（折半）
2. **介護保険料** - 40〜64歳のみ。標準報酬月額 × 料率 ÷ 2
3. **厚生年金保険料** - 標準報酬月額 × 18.3% ÷ 2
4. **雇用保険料** - 総支給額 × 被保険者料率
5. **所得税（源泉徴収）** - (総支給額 - 社保控除) を税額表に照合
6. **住民税** - 市区町村からの通知額をそのまま控除

### 重要: 料率はハードコード禁止
- 全ての料率・控除額はrate-master.jsonから動的に取得すること
- 改定時期: 健保(3月)、雇用保険(4月)、所得税(1月)、最低賃金(10月)

## 社会保険の仕組み

### 標準報酬月額
- 4〜6月の報酬平均で決定（算定基礎届）
- 等級表（1〜50等級）に当てはめる
- 年1回の定時決定 + 随時改定（月額変更届）

### 随時改定（月額変更届）の条件
以下の**3つ全て**を満たす場合:
1. 固定的賃金に変動があった
2. 変動月以降3ヶ月の報酬平均が2等級以上変動
3. 3ヶ月とも支払基礎日数が17日以上

### 賞与の社会保険料
- 標準賞与額（千円未満切り捨て）× 各保険料率
- 上限: 健保 年度累計573万円、厚年 1回150万円

## 年末調整

### スケジュール
- **10月**: 控除証明書の収集開始
- **11月**: 従業員から申告書回収
- **12月**: 年末調整計算、過不足精算
- **1月**: 源泉徴収票・給与支払報告書の提出

### 必要な申告書
1. 扶養控除等（異動）申告書
2. 基礎控除申告書兼配偶者控除等申告書兼年末調整に係る定額減税のための申告書
3. 保険料控除申告書
4. 住宅借入金等特別控除申告書（該当者のみ）

### 主な控除項目
- 基礎控除（58万円 ※令和8年〜）
- 配偶者控除・配偶者特別控除
- 扶養控除
- 特定親族特別控除（新設 ※令和8年〜）
- 生命保険料控除
- 地震保険料控除
- 社会保険料控除
- 住宅ローン控除

## 入退社手続き

### 入社時
1. 雇用契約書の締結
2. 社会保険（健保・厚年）資格取得届 → 年金事務所（5日以内）
3. 雇用保険 資格取得届 → ハローワーク（翌月10日まで）
4. マイナンバー収集
5. 給与振込口座の届出
6. 住民税の特別徴収切替届

### 退社時
1. 社会保険 資格喪失届 → 年金事務所（5日以内）
2. 雇用保険 資格喪失届 + 離職証明書 → ハローワーク（10日以内）
3. 源泉徴収票の発行
4. 住民税の異動届
5. 最終給与・退職金の計算

## e-Gov電子申請

### 対応手続き
- 社会保険: 資格取得届、資格喪失届、算定基礎届、月額変更届、賞与支払届
- 雇用保険: 資格取得届、資格喪失届、高年齢/育児/介護給付
- 労働保険: 年度更新

### API仕様
- 認証: OAuth2 / OpenID Connect
- データ形式: XML
- 検証環境: あり（テスト用電子証明書提供）
- ドキュメント: https://developer.e-gov.go.jp/

## 開発時の必須チェックリスト

- [ ] 料率はrate-master.jsonから動的取得しているか
- [ ] 端数処理は正しいか（50銭以下切捨て、50銭超切上げ）
- [ ] 非課税通勤手当を所得税計算から除外しているか
- [ ] 標準報酬月額の等級上下限を考慮しているか
- [ ] マイナンバーの暗号化・アクセス制御は適切か
- [ ] 36協定の上限（月45h/年360h、特別条項含む）を考慮しているか
- [ ] 最低賃金を下回る時給設定を防止しているか
