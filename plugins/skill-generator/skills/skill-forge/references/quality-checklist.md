# 品質チェックリスト (5層バリデーション)

## Layer 1: 構造検証

- [ ] `SKILL.md` が存在する（大文字小文字正確）
- [ ] フォルダ名が kebab-case
- [ ] フォルダ内に README.md がない
- [ ] オプション: scripts/, references/, assets/ のみ
- [ ] SKILL.md が500行以下（超過時は references/ に分離）

## Layer 2: 仕様準拠検証

- [ ] `---` デリミタが正しく配置
- [ ] `name`: kebab-case, スペースなし, 大文字なし, 64文字以内
- [ ] `description`: 空でない, 1024文字以内
- [ ] フロントマターにXMLタグ (`<` or `>`) なし
- [ ] name に "claude" "anthropic" を含まない
- [ ] `allowed-tools` の値が有効なツール名
- [ ] `model` の値が有効 (sonnet/opus/haiku/inherit)
- [ ] `context` の値が有効 (fork のみ)

## Layer 3: 品質検証

### Description品質
- [ ] WHAT: 何をするか記述されている
- [ ] WHEN: いつ使うか記述されている（トリガー条件）
- [ ] トリガーフレーズが具体的（ユーザーが言いそうな言葉）
- [ ] 関連ファイルタイプの言及（該当する場合）
- [ ] NOT条件（過剰トリガー防止、該当する場合）

### Content品質
- [ ] 明確なステップバイステップ指示がある
- [ ] コード例が含まれている（主要操作に対して）
- [ ] エラーハンドリングが含まれている
- [ ] トラブルシューティングセクションがある

### Progressive Disclosure
- [ ] コア指示が SKILL.md 内にある
- [ ] 詳細ドキュメントが references/ にある（必要な場合）
- [ ] 大きなコンテンツが別ファイルに分離されている
- [ ] references/ への明確なリンクがある

## Layer 4: Hostile Validation

### 単一責任原則
- [ ] スキルが1つの明確な責務のみを持つ
- [ ] 「AもBもCもする」という多機能スキルでない
- [ ] 名前が責務を正確に反映している

### 非ハルシネーション
- [ ] 存在しないツール/API/コマンドを参照していない
- [ ] 実際に動作するコード例のみ含まれている
- [ ] 架空のファイルパスを参照していない

### プレースホルダ拒否
- [ ] TODO / FIXME / TBD / PLACEHOLDER がない
- [ ] `...` や `etc.` で省略された指示がない
- [ ] 「ここに追加」のような未完成マーカーがない

### セキュリティ
- [ ] シークレット/APIキーのハードコードがない
- [ ] 危険なコマンド (rm -rf /, sudo 等) の無条件実行がない
- [ ] 外部URLへの無検証アクセスがない

## Layer 5: トリガーテスト

3つのテストプロンプトでトリガー精度を検証:

### テスト1: 正のトリガー
スキルが発動すべきプロンプトで発動するか

### テスト2: 正のトリガー（変形）
別の表現でも発動するか

### テスト3: 負のトリガー
発動すべきでないプロンプトで発動しないか

## 合格基準

| 層 | 合格条件 |
|----|---------|
| Layer 1 | 全項目クリア（必須） |
| Layer 2 | 全項目クリア（必須） |
| Layer 3 | 80%以上クリア |
| Layer 4 | 全項目クリア（必須） |
| Layer 5 | 2/3 テスト合格 |

**総合スコア 90点以上で合格**
