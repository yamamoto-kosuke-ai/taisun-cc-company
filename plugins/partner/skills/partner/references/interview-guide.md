# ヒアリングガイド

パートナーがユーザーの業務を深く理解するためのヒアリング設計。
質問は機械的に全て聞くのではなく、会話の流れで自然に深掘りする。

---

## ヒアリングの原則

1. **ユーザーの言葉で理解する** - 業界用語やシステム名をそのまま受け取り、わからなければ聞く
2. **具体例で確認する** - 「最近やった仕事を1つ教えてください」が最も有効
3. **認識の齟齬を防ぐ** - 曖昧な表現は必ず深掘りする（「資料」→どんな資料？誰向け？）
4. **押し付けない** - AIにできることを先に並べるのではなく、まず聞く
5. **5〜10往復で完了** - 長すぎるヒアリングは負担になる

---

## Phase 1: 仕事の全体像（1〜2往復）

### 最初の質問
```
はじめまして！あなたの業務パートナーになります。
まず、あなたのお仕事について教えてください。
どんな役割で、普段どんな業務をされていますか？
```

### 深掘りの方向
- 回答が抽象的（「営業です」）→ 「営業の中でも、新規開拓が多いですか？既存フォローが多いですか？」
- 回答が具体的（「クライアントへの提案書作成と進捗管理」）→ Phase 2へ進む
- 複数業務が出た → 「その中で、一番時間を使っているのはどれですか？」

---

## Phase 2: 具体的な業務の深掘り（3〜5往復）

### 核心の質問
```
具体的にイメージしたいので、最近やったお仕事を1つ教えてもらえますか？
例えば先週やった仕事とか、今まさに取り組んでいる仕事とか。
```

### Step A: 業務の概要を把握する

ユーザーが答えたら、まず全体像を確認する：

| 確認すべきこと | 聞き方の例 |
|-------------|-----------|
| **最終成果物** | 「それは最終的にどんな形になりますか？スライド？ドキュメント？」 |
| **誰に渡すか** | 「それは誰に見せるものですか？社内？クライアント？」 |
| **どこに置くか** | 「完成したらどこに保存しますか？Googleドライブ？Chatworkで送る？」 |
| **頻度** | 「それはどれくらいの頻度でやりますか？毎日？週1？」 |

### Step B: 作業手順を具体的に聞く（重要）

概要がわかったら、**その業務を最初から最後まで、どういう手順でやっているか**を聞く。
これがAIの活用ポイントを正確に特定するために最も重要なステップ。

```
その仕事をするとき、最初から最後まで
どんな手順で進めていますか？
最初に何をして、次に何をして...という流れを教えてください。
```

### 手順の深掘り例

ユーザーの回答が「調べて、まとめて、資料にする」のように粗い場合は、各ステップを具体化する：

```
「調べる」のところをもう少し詳しく教えてください。
- 何を使って調べますか？（Google検索？社内資料？誰かに聞く？）
- どんな情報を探しますか？
- だいたいどれくらい時間がかかりますか？
```

```
「まとめる」のところは具体的にどうしていますか？
- どこにメモしますか？（いきなりスライド？一旦メモ帳？）
- 構成やフォーマットは決まっていますか？テンプレートがある？
- 過去の資料を参考にすることはありますか？
```

```
「資料にする」ところですが、
- Googleスライドで作りますか？他のツール？
- 図やグラフは入れますか？どうやって作りますか？
- 完成後、誰かに確認を出しますか？フィードバックのやり取りは？
- 最終的にどうやって相手に届けますか？
```

### 手順の記録フォーマット

ヒアリングで把握した手順は、以下の形式で内部的に整理する：

```
業務: [業務名]
頻度: [毎日/週X回/月X回/不定期]
所要時間: [全体でXX分〜XX時間]

手順:
1. [ステップ名] (XX分)
   - やること: [具体的な作業内容]
   - 使うツール: [ツール名]
   - インプット: [何を見る/参照する]
   - アウトプット: [何ができる]
   → AI代替可能性: [高/中/低/不可]

2. [ステップ名] (XX分)
   ...

ボトルネック: [一番時間/手間がかかるステップ]
```

### Step C: ボトルネックの特定

手順が出揃ったら、ペインポイントを確認する：

```
なるほど、一通りの流れが見えました。
この中で、一番面倒だったり時間がかかるのはどこですか？
```

追加で聞けると良いこと：
- 「ミスしやすいポイントはありますか？」
- 「やり直しになることはありますか？どんなとき？」
- 「この作業を誰かに引き継ぐとしたら、何が一番教えにくいですか？」

### 認識齟齬チェック

手順まで含めて要約し、確認する：

```
ここまでの理解を確認させてください。

〇〇さんの提案資料作成の流れ：
1. まず競合のWebサイトをGoogle検索で調べる（30分〜1時間）
2. 情報をGoogleドキュメントにメモする
3. 過去の提案資料を参考に構成を決める
4. Googleスライドで資料を作成（1〜2時間）
5. 上司に確認 → フィードバック反映
6. Chatworkでクライアントに共有

一番時間がかかるのは 1. の調査と 4. のスライド作成。

合ってますか？違うところがあれば教えてください。
```

## Phase 3: 他の業務パターン（2〜3往復）

### 広げる質問
```
ありがとうございます。他にも定期的にやっている仕事はありますか？
例えば：
- 定例の報告や会議の準備
- 情報収集やリサーチ
- データの整理や集計
- 社内外への連絡・調整
```

### 2つ目以降の業務も手順を聞く

メイン業務ほど詳細でなくてよいが、**ざっくりとした手順とボトルネック**は確認する：

```
その仕事もざっくりで構いません、
どんな手順で進めていますか？
あと、面倒なポイントがあれば教えてください。
```

手順が3ステップ以上出たら「十分です」と切り上げてOK。
ポイントは**全ての業務でAI活用の接点を見つけること**。

### 業務一覧の整理

ヒアリングした業務は一覧化して確認する：

```
ここまでで出てきたお仕事を整理すると：

1. 提案資料の作成（週2〜3回、一番時間かかる）
   → 調査→メモ→構成→スライド作成→確認→共有
2. 週次の進捗報告（毎週月曜）
   → 各案件の状況確認→報告書作成→Chatworkに投稿
3. クライアントへのメール対応（毎日）
   → 受信確認→返信作成→上司に相談することも

他にもありますか？なければ次に進みます。
```

## Phase 4: ツール環境の確認（1往復）

### 確認する項目
```
使っているツールも確認させてください。
主にどれを使っていますか？
- Chatwork（メッセージ・ファイル共有）
- Googleスライド / ドキュメント / スプレッドシート
- Googleドライブ
- その他のツール
```

### ツール連携の検出
この時点で、プロジェクトルートに以下のファイルがあるか自動チェック:
- サービスアカウントJSONキー（*.json でキーファイルっぽいもの）
- `.chatwork-config.json`

検出結果を報告:
```
確認しました。
- Google連携: ✅ 利用可能（サービスアカウントキー検出）
- Chatwork連携: ❌ 未設定（.chatwork-config.json が見つかりません）
  → 設定方法をお伝えしましょうか？
```

## Phase 5: AI活用マップの提示とプロファイル保存（1〜2往復）

### 業務×AI活用の提案

ヒアリングで把握した**具体的な手順**に沿って、どのステップをAIが代替・加速できるか提案する。
スキル名やエージェント名は出さず、**「あなたの〇〇の作業のうち、△△の部分」**という形で伝える。

```
ヒアリングありがとうございました。
あなたのお仕事の流れに沿って、私がお手伝いできるところを整理しました：

【提案資料の作成】
 1. 競合調査（今: Google検索で30分〜1時間）
    → お任せください。競合のサービス・価格・特徴を一括で調べて
      比較表にまとめます。スプレッドシートにも出力できます。
 2. 構成を考える（今: 過去資料を見ながら30分）
    → 調査結果をもとに構成案を提案します。
      過去の構成パターンを教えてもらえれば、それに合わせます。
 3. スライド作成（今: 1〜2時間）
    → 構成案からGoogleスライドに直接書き出せます。
      デザインの調整は手動になりますが、テキストと構造は任せてください。
 4. 確認依頼の送信（今: Chatworkで手動）
    → 確認依頼のメッセージを作成してChatworkに送れます。

【週次報告】
 1. 各案件の状況まとめ（今: 各所から情報を集めて30分）
    → 前回の報告内容を覚えているので、差分を聞くだけでまとめます。
 2. 報告文の作成・投稿（今: 15分）
    → Chatworkの記法に合わせて作成、そのまま送信もできます。

まずは「提案資料の競合調査」あたりから試してみませんか？
```

**重要**: 提案は必ずユーザーの実際の手順に紐づける。
「こんなことができます」ではなく「あなたのこの作業のここを代わりにやります」。

### プロファイル保存
```
プロファイルを保存しました。
次回以降はヒアリングなしで、すぐお手伝いできます。
何か変わったことがあれば「プロファイル更新して」と言ってください。
```

## ヒアリングのNG集

| NG | なぜNG | 代わりに |
|----|-------|---------|
| 「何をお手伝いしましょうか？」 | 何ができるか知らないので答えられない | 「どんなお仕事をされていますか？」 |
| 「mega-researchで調査できます」 | スキル名は意味不明 | 「競合の情報を調べてまとめられます」 |
| 「どの部署を使いたいですか？」 | 部署は内部概念 | 質問しない。自動判断する |
| 20個の質問を一気に聞く | 疲れる | 会話の流れで自然に |
| 「AIにできることリスト」を最初に見せる | ピンとこない | まず仕事を聞いて、マッチするものだけ提示 |

## 業種別ヒアリングのヒント

ユーザーの回答から業種が特定できた場合の追加質問：

### 営業系
- 「提案書はどんな構成ですか？テンプレートはありますか？」
- 「商談後の報告はどこにまとめていますか？」
- 「リード情報の管理はどうしていますか？」

### 企画・マーケ系
- 「コンテンツはどのプラットフォーム向けが多いですか？」
- 「KPIの集計はどうやっていますか？」
- 「競合分析はどれくらいの頻度でやりますか？」

### 事務・管理系
- 「定型的な報告書やフォーマットはありますか？」
- 「データの集計や転記の作業はありますか？」
- 「社内への周知連絡はどこで行いますか？」

### 制作・クリエイティブ系
- 「ブリーフやオリエンの情報はどこからもらいますか？」
- 「制作物の確認・フィードバックのやり取りはどこでしますか？」
- 「素材やアセットの管理はどうしていますか？」

### 技術系
- 「ドキュメントはどこに書いていますか？」
- 「コードレビューのプロセスはありますか？」
- 「技術選定のリサーチはどうやっていますか？」
